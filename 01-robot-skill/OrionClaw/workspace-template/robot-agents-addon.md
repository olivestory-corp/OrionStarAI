## 🤖 机器人配置（每个 session 必读）

**任何 session 都可以控制这台机器人！**

- **连接方式**：读取 `skills/robot-control/SKILL.md`，里面有完整的 44+ 条指令说明

### 🗺️ 关于"地图"的说明

机器人的"地图"指的是**导航点位列表**，不是图片。需要通过命令实时获取：

```bash
curl -s -X POST "http://<GATEWAY_IP>:18795/robot/cmd?token=<YOUR_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"deviceId":"<YOUR_DEVICE_ID>","cmd":"robot.getPlaceList","args":{},"timeoutMs":20000}'
```

- 返回当前地图上所有已标记的命名点位
- **导航前必须先调此接口**，用真实名字，不能猜测
- ⚠️ 地图随时可能切换，不要把点位名称硬编码，每次用前实时获取

### 📸 拍照注意事项

拍照前需注意机器人头部角度：

1. **先低头拍一张**，看清楚实际画面内容
2. **根据内容判断角度是否合适**，如需要再调整（抬头/转向）重拍
3. 拍完后执行 `head.reset` 恢复默认角度

### ⚠️ 机器人任务进度播报（强制规则，无例外）

执行任何机器人任务时，**每完成一个子步骤后，必须立即通过 `message` 工具向用户发送进度播报**。

- 格式：`[emoji] + 简短描述（≤30字）`
- 时机：每条机器人指令返回后立刻发，无论成功还是失败
- 示例：
  - `🤖 收到任务！正前往目的地…`
  - `📍 已到达目标地点`
  - `✅ 拍照完成，正在上传…`
  - `❌ 导航失败，正在重试`
- **禁止**：执行完所有步骤后只发一条总结。每步都要播报！

### ⚠️ 图片传递规则（强制，无例外）

**永远不要使用 base64 传递图片数据！**

- 图片必须通过文件路径或 URL 传递
- 拍照后解码 base64 保存为本地文件，再用 `message` 工具发送：

```
message(action=send, media="/path/to/photo.jpg")
```
